    What “Makebakery” means

    Michael F. Lamb

    Sun, 11 Feb 2018 13:25:15 -0800

    Makebakery used to be “m4-bakery,” because it grew out of a simpler first iteration that also used GNU Make but focused on GNU m4, another ubiquitous old UNIX tool used for templating and macro expansion.

    The article Using M4 to write HTML (1997)1 was my original inspiration to build it that way.

    However, the learning curve for m4 is very steep and the resulting code can be quite ugly, especially for nontrivial tasks.

    Now, all the m4-related logic is relegated to a module and not enabled by default, so you can easily avoid it in favor of more modern template mechanisms, or none at all.

    The “bakery” in Makebakery references a 2002 blog post by the late Aaron Swartz2:

    Some websites, the webmake documentation explains, are fried up for the user every time. But others are baked once and served up again and again.

    This references the documentation for WebMake3, an old perl-based static website generator, and that references Baking Versus Frying, a slide in a presentation by Ian Kallen at OSCon 20004, which seems to have originated this use of the term.
